## 0. OpenClaw原典の理解

- Channels: <https://docs.openclaw.ai/channels>
- Gateway: <https://docs.openclaw.ai/gateway>

OpenClawは複数チャネルを同一Gateway配下で運用し、チャネル差分をアダプタに閉じ込める設計。

## 3. 契約仕様

### 3.1 共通イベント

```go
type ChannelEvent struct {
    Channel   string // line|telegram|discord|slack
    ChatID    string
    UserID    string
    MessageID string
    Text      string
    Timestamp time.Time
    Raw       []byte
}
```

### 3.2 Adapter I/F

```go
type ChannelAdapter interface {
    Name() string
    Start(ctx context.Context) error
    Stop(ctx context.Context) error
    Send(ctx context.Context, chatID string, text string) error
    Verify(req *http.Request, body []byte) error
}
```

### 3.3 セッション規約

- 形式: `YYYYMMDD-{channel}-{tenant?}-{chat}`
- 1日1セッション境界は既存方針を踏襲。
