**作成日**: 2026-03-07

**ベース**: v3.0（Clean Architecture）
**ステータス**: 実装予定

---

## 0. エグゼクティブサマリ

### 0.1 目的

ユーザーが Mio に「もっとカジュアルに喋って」「敬語やめて」等と指示すると、
Mio が自分自身のペルソナ設定ファイル (`workspace/CHAT_PERSONA.md`) を書き換える。

### 0.2 要件

| 要件 | 詳細 |
|------|------|
| 自律的編集 | ユーザーの自然言語指示で Mio がペルソナファイルを書き換える |
| 既存パターン踏襲 | Web検索と同じキーワードベース検出 + 2段階処理 |
| 安全性 | 書き換え前の内容をログ出力。ファイル構造（マークダウン）は維持 |
| 確認応答 | 書き換え後、Mio が変更内容をユーザーに報告 |

### 0.3 スコープ外

- `SOUL.md`, `IDENTITY.md` の編集（v1 では `CHAT_PERSONA.md` のみ）
- 編集履歴の永続化（Git auto-commit で代替）
- 承認フロー（v1 ではユーザー指示を即反映）

## 1. アーキテクチャ

### 1.1 コンポーネント構成

```
User: "もっとカジュアルに喋って"
         |
    MioAgent.Chat()
         |
    キーワード検出（ペルソナ調整意図）
         |
    PersonaEditor.Read() → 現在のペルソナ取得
         |
    LLM.Generate() → 新しいペルソナ生成
         |
    PersonaEditor.Write() → ファイル書き換え
         |
    確認メッセージ返却
```

### 1.2 パッケージ配置（Clean Architecture 準拠）

| レイヤー | パッケージ | 責務 |
|---------|-----------|------|
| Domain | `internal/domain/agent` | `PersonaEditor` インターフェース定義 |
| Infrastructure | `internal/infrastructure/persona` | ファイルベースの `PersonaEditor` 実装 |
| Composition | `cmd/picoclaw` | DI（PersonaEditor を MioAgent に注入） |

### 1.3 依存方向

```
domain/agent (PersonaEditor interface)
     ^
     |
infrastructure/persona (FilePersonaEditor)
     ^
     |
cmd/picoclaw (DI wiring)
```

## 4. 検出ロジック

### 4.1 トリガーキーワード

以下のキーワードのいずれかがユーザーメッセージに含まれる場合、ペルソナ調整意図と判定:

```go
personaKeywords := []string{
    "ペルソナ", "キャラ", "口調", "語尾", "喋り方", "話し方",
    "敬語", "タメ口", "カジュアル", "フォーマル",
    "もっと優しく", "もっと厳しく", "テンション",
}
```

AND 条件: 以下の動詞系キーワードのいずれかも含む:

```go
personaActionKeywords := []string{
    "変えて", "にして", "やめて", "直して", "調整", "修正",
    "書き換え", "編集", "更新",
}
```

### 4.2 判定ロジック

```
ペルソナ調整意図 = (personaKeywords のいずれかを含む) AND (personaActionKeywords のいずれかを含む)
```

両方のキーワード群にマッチした場合のみ発火。片方だけでは通常の会話として処理。

## 5. 処理フロー

### 5.1 ペルソナ編集の2段階処理

**Step 1**: 現在のペルソナを読み込み、LLM に新しいペルソナを生成させる

```go
prompt := fmt.Sprintf(
    "以下は現在のペルソナ設定です:\n\n%s\n\n"+
    "ユーザーの要求: %s\n\n"+
    "上記の要求に基づいて、ペルソナ設定を書き換えてください。"+
    "形式（マークダウン）と基本構造は維持してください。"+
    "書き換えた設定のみを出力してください。説明は不要です。",
    currentPersona, userMessage,
)
```

**Step 2**: 生成結果をファイルに書き込み、確認メッセージを返す

### 5.2 エラーハンドリング

- `PersonaEditor` が nil → 通常の会話として処理（フォールバック）
- ファイル読み込み失敗 → エラーメッセージを返却
- LLM 生成失敗 → エラーメッセージを返却
- ファイル書き込み失敗 → エラーメッセージを返却

## 6. 安全性

### 6.1 ログ出力

書き換え前後の内容を `log.Printf` で記録:
```
[Mio] Persona edit requested: "もっとカジュアルに喋って"
[Mio] Persona before: (先頭100文字)
[Mio] Persona after: (先頭100文字)
```

### 6.2 制約

- 編集対象は `CHAT_PERSONA.md` のみ（`SOUL.md`, `IDENTITY.md` は不可）
- 空文字列への書き換えは拒否
- ファイルサイズ上限: 4KB（異常な肥大化防止）
