## 1. エージェント構成

### コアエージェント（合計5体）

-   Chat
-   Worker
-   Order1（DeepSeek R1.5）
-   Order2（OpenAI API / ChatGPT）
-   Order3（Anthropic Cloud / SSHブリッジ）

すべての作業には **JobID を付与することを必須** とする。\
すべての応答は対応する **JobID を明示することを必須** とする。

------------------------------------------------------------------------

## 2. 役割定義

### Chat

-   会話インターフェース専用
-   重い処理は行わない
-   タスク結果を受領した時点で終了

### Worker

-   データ収集
-   Webクローリング
-   Heartbeatの集約管理
-   すべてのエージェント状態をChatへ報告

### Order1

-   低コスト・大量処理向け
-   比較的単純な作業を担当
-   DeepSeek R1.5

### Order2

-   コーディングおよび実装の中核
-   OpenAI API（ChatGPT）

### Order3

-   SSHブリッジ経由で別PCへ接続
-   リモート環境のLLMを利用
-   将来的な拡張ブリッジ基盤

------------------------------------------------------------------------

## 3. SSHブリッジ設計

-   ブリッジモジュールは共通化
-   接続先のみ切り替え可能とする
-   Folder1 / Folder2 / Folder3 / 将来のFolder4--6へ拡張可能
-   横方向スケールを前提とした構造

------------------------------------------------------------------------

## 4. Heartbeatシステム

すべてのエージェントは： - 自律的にHeartbeatを発行する -
動作状態をWorkerへ報告する

Workerは： - すべてのHeartbeatを集約する - 自身のHeartbeatも含める -
Chatへ統合状態を報告する

------------------------------------------------------------------------

## 5. 合議制（Deliberation Mode）

必要時に以下を実行可能とする：

1.  各Coderが独立して提案を生成
2.  Workerが提案を集約
3.  擦り合わせ・比較
4.  Chatが最終決定

Spawn多用を避けつつ協調動作を実現する方式。

------------------------------------------------------------------------

## 6. アイドル（Idle）知性モード

タスク未保有エージェントのみ： - ブレインストーミングを実施可能 -
システム改善提案を生成可能 - 実行中タスクへ干渉してはならない

------------------------------------------------------------------------

## 7. メモリ制約前提

-   想定環境：8GB RAM
-   Spawn多発構成は避ける
-   固定メンバー協調型モデルを採用

------------------------------------------------------------------------

システム知性の向上
