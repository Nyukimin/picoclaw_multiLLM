## 0. エグゼクティブサマリ

### 0.1 目的

Agent 間のメッセージフローをブラウザでリアルタイム監視する。
LINE が使えない場合でも、ブラウザからメッセージを送信して Chat（Mio）と会話できる。

### 0.2 要件

| 要件 | 詳細 |
|------|------|
| リアルタイム表示 | SSE (Server-Sent Events) でブラウザにイベントをストリーム |
| メッセージ送信 | ブラウザの入力欄から Mio にメッセージを送れる |
| v3/v4 両対応 | Local Mode (v3) / Distributed Mode (v4) の両方で動作する |
| 単一ページ | HTML/CSS/JS を1ファイルに完結。外部依存なし |
| 認証 | v1.0 では Tailscale ACL に委任（パブリック非公開） |

### 0.3 表示するイベント

| イベント種別 | 発火タイミング | 表示 |
|-------------|---------------|------|
| `message.received` | ユーザーメッセージ受信時 | User -> Mio のメッセージ |
| `routing.decision` | Mio がルーティング決定時 | システムバッジ（ルート名 + confidence） |
| `agent.start` | Agent が処理開始時 | 「考え中...」「Coder3に依頼」等 |
| `agent.response` | Agent が応答返却時 | 応答内容 |

### 0.4 会話フローの可視化

ビュワーは Agent 間の会話を方向付きで表示する。

**CHAT ルート:**
```
User -> Mio: ユーザーメッセージ
── routing: CHAT ──
Mio -> User: 応答
```

**CODE3 ルート（v3 Local Mode）:**
```
User -> Mio: ユーザーメッセージ
── routing: CODE3 ──
Mio -> Shiro: 「Coder3にコード生成を依頼」
Shiro -> Claude: タスク内容
Claude -> Shiro: Plan + Patch
Shiro -> Mio: 実行結果
```

**CODE3 ルート（v4 Distributed Mode）:**
```
User -> Mio: ユーザーメッセージ
── routing: CODE3 ──
Mio -> Coder3: タスク送信（SSH経由）
Coder3 -> Mio: 応答受信（SSH経由）
```

## 1. アーキテクチャ

### 1.1 コンポーネント構成

```
Browser <-- SSE --> /viewer/events (EventHub)
Browser  -- POST -> /viewer/send  (MessageHandler)
                          |
                    Orchestrator.ProcessMessage()
                          |
                    EventListener.OnEvent()
                          |
                      EventHub.broadcast()
                          |
                    SSE -> Browser
```

### 1.2 パッケージ配置（Clean Architecture 準拠）

| レイヤー | パッケージ | 責務 |
|---------|-----------|------|
| Application | `internal/application/orchestrator` | `EventListener` インターフェース、`OrchestratorEvent` 型 |
| Adapter | `internal/adapter/viewer` | `EventHub`（SSEブロードキャスター）、HTTP ハンドラ、HTML ページ |

### 1.3 依存方向

```
adapter/viewer --> application/orchestrator  (EventListener インターフェース)
cmd/picoclaw   --> adapter/viewer            (エンドポイント登録)
cmd/picoclaw   --> application/orchestrator  (Orchestrator にリスナー注入)
```

## 2. Application 層: EventListener

### 2.1 インターフェース定義

```go
// internal/application/orchestrator/event.go

type EventListener interface {
    OnEvent(ev OrchestratorEvent)
}

type OrchestratorEvent struct {
    Type      string `json:"type"`
    From      string `json:"from"`
    To        string `json:"to,omitempty"`
    Content   string `json:"content"`
    Route     string `json:"route,omitempty"`
    JobID     string `json:"job_id,omitempty"`
    Timestamp string `json:"timestamp"`
}
```

### 2.2 イベント発火箇所

**MessageOrchestrator（v3 Local Mode）:**

| メソッド | イベント | from | to | 内容 |
|---------|---------|------|-----|------|
| `ProcessMessage` 冒頭 | `message.received` | user | mio | ユーザーメッセージ |
| `ProcessMessage` ルーティング後 | `routing.decision` | mio | - | ルート + confidence |
| `executeTask` 各ルート開始時 | `agent.start` | 委任元 | 委任先 | 処理開始通知 |
| `executeTask` 各ルート完了時 | `agent.response` | 実行Agent | 報告先 | 応答内容 |

**DistributedOrchestrator（v4 Distributed Mode）:**

| メソッド | イベント | from | to | 内容 |
|---------|---------|------|-----|------|
| `ProcessMessage` 冒頭 | `message.received` | user | mio | ユーザーメッセージ |
| `ProcessMessage` ルーティング後 | `routing.decision` | mio | - | ルート + confidence |
| `executeDistributed` ローカル処理時 | `agent.start` / `agent.response` | mio | user | Chat 応答 |
| `executeDistributed` リモート送信時 | `agent.start` | mio | targetAgent | タスク送信 |
| `executeViaSSH` / `executeViaLocal` 応答受信時 | `agent.response` | targetAgent | mio | 応答内容 |

## 3. Adapter 層: EventHub + HTTP ハンドラ

### 3.1 EventHub

- `orchestrator.EventListener` を実装
- 接続中の SSE クライアントにイベントをファンアウト
- 直近 200 件のイベント履歴を保持（新規接続時に送信）

### 3.2 HTTP エンドポイント

| パス | メソッド | 説明 |
|------|---------|------|
| `/viewer` | GET | HTML ページ配信 |
| `/viewer/events` | GET (SSE) | イベントストリーム |
| `/viewer/send` | POST | メッセージ送信 |

### 3.3 `/viewer/send` リクエスト/レスポンス

```json
// Request
POST /viewer/send
{"message": "こんにちは"}

// Response
{"ok": true}
```

メッセージは非同期で `Orchestrator.ProcessMessage()` に渡される。
結果は SSE 経由でブラウザに返る。

### 3.4 ProcessMessage パラメータ（Viewer 経由）

| フィールド | 値 |
|-----------|-----|
| SessionID | `"viewer"` |
| Channel | `"viewer"` |
| ChatID | `"viewer-user"` |
| UserMessage | ユーザー入力テキスト |

## 4. HTML/CSS/JS 仕様

### 4.1 ファイル

`internal/adapter/viewer/viewer.html`（`go:embed` でバイナリに埋め込み）

### 4.2 デザイン

- ダークテーマ（背景 `#0b0b14`）
- Agent 別カラー: Mio(#f472b6), Shiro(#22d3ee), Coder1(#fb923c), Coder2(#818cf8), Coder3(#a78bfa)
- メッセージはアバター + 名前 + 方向 + タイムスタンプ + 内容
- ルーティング決定はシステムバッジ（中央区切り線）
- 入力欄はページ下部固定（Enter で送信、Shift+Enter で改行）

### 4.3 SSE 再接続

切断時は 3 秒後に自動再接続。接続状態はヘッダー右上のインジケーターに表示。

## 5. 制約と今後

### 5.1 v1.0 の制約

- 認証なし（Tailscale ACL に依存）
- セッションは `"viewer"` 固定（複数ブラウザで共有）
- v4 Distributed Mode のリモート Agent 内部イベントは見えない（送信/受信のみ）

### 5.2 将来拡張（v2.0 候補）

- Basic 認証 or Tailscale Whois による認証
- セッション分離（ブラウザ別）
- Agent 別フィルタリング
- イベントの永続化（ファイル or DB）
