## 1. 概要

### 1.1 目的

LINE 以外のチャットプラットフォーム（Discord, Slack）および音声入出力を
adapter 層に追加し、Orchestrator 以下のドメインロジックは一切変更しない。

### 1.2 設計原則

- **Orchestrator は Channel を知らない**: `ProcessMessageRequest.Channel` フィールドが変わるだけ
- **adapter 層で完結**: 各プラットフォームの認証・受信・送信を adapter 内に閉じる
- **infrastructure は共有**: LLM, Session, Routing 等はチャネル間で共有

### 1.3 追加するチャネル一覧

| チャネル | 通信方式 | 優先度 | 依存ライブラリ |
|---------|---------|--------|--------------|
| Discord | WebSocket Gateway | B | `github.com/bwmarrin/discordgo` |
| Slack | Socket Mode | B | `github.com/slack-go/slack` |
| 音声 | ローカルデバイス / WebSocket | C | Whisper (STT), VOICEVOX/Edge TTS (TTS) |

## 2. 現行アーキテクチャ分析

### 2.1 LINE アダプターの構造（参照実装）

```
internal/adapter/line/
├── handler.go       # HTTP webhook受信 → ProcessMessageRequest変換 → Orchestrator呼出
├── sender.go        # LINE Reply/Push API呼出
├── signature.go     # HMAC-SHA256 署名検証
└── media.go         # メディアダウンロード
```

**LINE Handler の責務**:
1. HTTP POST `/webhook` を受信
2. `X-Line-Signature` で HMAC-SHA256 署名検証
3. JSON ペイロードをパース → イベント抽出
4. テキストメッセージのみフィルタ（画像等はスキップ）
5. Group chat の場合は Bot mention チェック
6. `ProcessMessageRequest` を構築し `Orchestrator.ProcessMessage()` を呼出
7. レスポンスを LINE Reply API で送信

**キーポイント**: Handler は `Orchestrator` インターフェース（`ProcessMessage` 1メソッド）のみに依存。

### 2.2 Orchestrator インターフェース

```go
// internal/adapter/line/handler.go:15-17（現行）
type Orchestrator interface {
    ProcessMessage(ctx context.Context, req orchestrator.ProcessMessageRequest) (orchestrator.ProcessMessageResponse, error)
}
```

```go
// internal/application/orchestrator/message_orchestrator.go:17-22
type ProcessMessageRequest struct {
    SessionID   string
    Channel     string   // "line", "discord", "slack", "voice"
    ChatID      string   // プラットフォーム固有のユーザー/チャネルID
    UserMessage string
}
```

この構造は既にチャネル非依存。新チャネルは adapter 層の追加だけで対応可能。

## 3. 共通インターフェース設計

### 3.1 ChannelAdapter インターフェース

```go
// internal/adapter/channel.go
package adapter

import "context"

// ChannelAdapter はチャットプラットフォームとの統合を抽象化する。
// 各チャネル（LINE, Discord, Slack, Voice）はこのインターフェースを実装する。
type ChannelAdapter interface {
    // Name はチャネル識別子を返す（"line", "discord", "slack", "voice"）
    Name() string

    // Start はチャネルのリスナーを起動する（ブロッキング）。
    // ctx がキャンセルされたら graceful shutdown する。
    Start(ctx context.Context) error

    // Stop はチャネルのリスナーを停止する。
    Stop() error
}
```

**注意**: `SendMessage` は ChannelAdapter に含めない。
各チャネルの送信方式が異なるため（LINE は ReplyToken、Discord は ChannelID、Slack は thread_ts）、
送信は各 Handler の `processEvent` 内で直接呼び出す。

### 3.2 ResponseSender インターフェース（オプション）

Push 通知（アイドルチャット等）用に、チャネル横断の送信インターフェースを定義する。

```go
// internal/adapter/channel.go
type ResponseSender interface {
    // SendMessage はチャネルにメッセージを送信する（Push型）。
    // chatID のフォーマットはチャネルごとに異なる。
    SendMessage(ctx context.Context, chatID string, message string) error
}
```

| チャネル | chatID フォーマット | 送信方式 |
|---------|-------------------|---------|
| LINE | LINE UserID (`U...`) | Push API |
| Discord | `guildID:channelID` | Channel message send |
| Slack | `teamID:channelID:threadTS` | chat.postMessage |

### 3.3 ファイルレイアウト

```
internal/adapter/
├── channel.go                    # ChannelAdapter, ResponseSender インターフェース
├── line/                         # 既存（変更なし）
│   ├── handler.go
│   ├── sender.go
│   ├── signature.go
│   └── media.go
├── discord/                      # 新規
│   ├── handler.go                # WebSocket Gateway 受信 → Orchestrator
│   └── sender.go                 # Discord メッセージ送信
├── slack/                        # 新規
│   ├── handler.go                # Socket Mode 受信 → Orchestrator
│   └── sender.go                 # Slack メッセージ送信
└── voice/                        # 新規
    ├── handler.go                # 音声入力 → STT → Orchestrator → TTS → 音声出力
    ├── transcriber.go            # STT インターフェース + Whisper 実装
    └── synthesizer.go            # TTS インターフェース + VOICEVOX/EdgeTTS 実装
```

### 4.3 セッション ID 生成

```go
// DM: YYYYMMDD-discord-dm-{userID}
// サーバー: YYYYMMDD-discord-{guildID}-{channelID}
func (h *Handler) generateSessionID(guildID, channelID, userID string) string {
    datePrefix := time.Now().Format("20060102")
    if guildID == "" {
        return fmt.Sprintf("%s-discord-dm-%s", datePrefix, userID)
    }
    return fmt.Sprintf("%s-discord-%s-%s", datePrefix, guildID, channelID)
}
```

### 4.4 メンション除去

```go
// "<@BOT_ID> こんにちは" → "こんにちは"
func stripMention(content string, botUserID string) string {
    mention := fmt.Sprintf("<@%s>", botUserID)
    content = strings.Replace(content, mention, "", 1)
    return strings.TrimSpace(content)
}
```

| 項目 | 対応 |

## 5. Slack アダプター

### 5.1 構造

```go
// internal/adapter/slack/handler.go
package slack

type Handler struct {
    orchestrator Orchestrator
    client       *slack.Client
    socketMode   *socketmode.Client
    botUserID    string
}

func NewHandler(orch Orchestrator, botToken, appToken string) (*Handler, error)
func (h *Handler) Name() string
func (h *Handler) Start(ctx context.Context) error
func (h *Handler) Stop() error
```

### 5.2 通信方式: Socket Mode

Socket Mode を採用する理由:
- **Webhook 不要**: パブリック URL が不要（Tailscale 環境に最適）
- **Firewall フレンドリー**: アウトバウンド WebSocket のみ
- **LINE との対称性**: LINE は Webhook だが、Slack は Socket Mode で同等の即時性

### 5.3 メッセージ受信フロー

```
Slack Socket Mode (WebSocket)
    │
    ▼
socketmode.EventsAPIEvent
    │
    ├─ event_callback: message
    │   ├─ Bot自身のメッセージ → スキップ
    │   ├─ DM → 常に処理
    │   ├─ チャネル → Bot mention チェック
    │   │
    │   ▼
    │   ProcessMessageRequest {
    │       SessionID:   generateSessionID(teamID, channelID, threadTS)
    │       Channel:     "slack"
    │       ChatID:      userID
    │       UserMessage: text（メンション除去済み）
    │   }
    │   │
    │   ▼
    │   Orchestrator.ProcessMessage()
    │   │
    │   ▼
    │   chat.postMessage(channel, text, thread_ts)  // スレッド内返信
    │
    ├─ app_mention → 上記と同じフロー
    └─ その他 → スキップ
```

### 5.4 セッション ID 生成

```go
// スレッドあり: YYYYMMDD-slack-{teamID}-{channelID}-{threadTS}
// スレッドなし: YYYYMMDD-slack-{teamID}-{channelID}
// DM:          YYYYMMDD-slack-dm-{userID}
func (h *Handler) generateSessionID(teamID, channelID, threadTS, userID string, isDM bool) string {
    datePrefix := time.Now().Format("20060102")
    if isDM {
        return fmt.Sprintf("%s-slack-dm-%s", datePrefix, userID)
    }
    if threadTS != "" {
        return fmt.Sprintf("%s-slack-%s-%s-%s", datePrefix, teamID, channelID, threadTS)
    }
    return fmt.Sprintf("%s-slack-%s-%s", datePrefix, teamID, channelID)
}
```

### 5.5 Slack 固有の考慮事項

| 項目 | 対応 |
|------|------|
| メッセージ長制限 | 4000文字（Block Kit で拡張可能） |
| スレッド | `thread_ts` でスレッド内返信。セッション管理と自然にマッピング |
| メンション除去 | `<@U12345>` パターンを除去 |
| Block Kit | 将来的にリッチメッセージ（コードブロック、ボタン等）対応 |
| App Token | Socket Mode に必要。`xapp-` プレフィックス |
| Bot Token | API 呼出に必要。`xoxb-` プレフィックス |
| Event subscription | `message.channels`, `message.groups`, `message.im`, `app_mention` |

## 6. 音声入出力アダプター

### 6.1 アーキテクチャ

```
マイク / WebSocket
    │
    ▼
┌──────────────┐
│ Transcriber  │  STT: 音声 → テキスト
│ (Whisper)    │
└──────┬───────┘
       │ text
       ▼
ProcessMessageRequest {
    Channel: "voice"
    UserMessage: transcribed text
}
       │
       ▼
Orchestrator.ProcessMessage()
       │
       ▼ response text
┌──────────────┐
│ Synthesizer  │  TTS: テキスト → 音声
│ (VOICEVOX)   │
└──────┬───────┘
       │ audio bytes
       ▼
スピーカー / WebSocket
```

### 6.2 domain インターフェース

```go
// internal/domain/voice/transcriber.go
package voice

import "context"

// Transcriber は音声をテキストに変換する（STT）。
type Transcriber interface {
    Transcribe(ctx context.Context, audio []byte, format AudioFormat) (string, error)
}

// AudioFormat は音声フォーマットを表す。
type AudioFormat string

const (
    FormatWAV  AudioFormat = "wav"
    FormatMP3  AudioFormat = "mp3"
    FormatOGG  AudioFormat = "ogg"
    FormatWebM AudioFormat = "webm"
)
```

```go
// internal/domain/voice/synthesizer.go
package voice

import "context"

// Synthesizer はテキストを音声に変換する（TTS）。
type Synthesizer interface {
    Synthesize(ctx context.Context, text string, opts SynthesizeOptions) ([]byte, error)
}

// SynthesizeOptions はTTSのオプション。
type SynthesizeOptions struct {
    Voice  string      // 話者名/ID
    Format AudioFormat // 出力フォーマット
    Speed  float64     // 話速（1.0 = 標準）
}
```

### 6.5 Voice の動作モード

| モード | 入力 | 出力 | ユースケース |
|--------|------|------|------------|
| ローカル | マイク (portaudio) | スピーカー (portaudio) | ラズパイ等のデバイス |
| WebSocket | WebSocket audio stream | WebSocket audio stream | ブラウザ/リモートクライアント |
| チャネル連携 | LINE/Discord 音声メッセージ | 音声ファイル返信 | 既存チャネルでの音声対応 |

**チャネル連携モード**の場合、Voice Handler は独立して動作するのではなく、
各チャネルの Handler 内で Transcriber/Synthesizer を呼び出す:

```go
// 例: LINE Handler での音声メッセージ対応（将来）
func (h *Handler) processAudioEvent(event WebhookEvent) {
    // 1. 音声ファイルダウンロード
    audio, _ := h.mediaDownloader.Download(ctx, event.Message.ID)

    // 2. STT
    text, _ := h.transcriber.Transcribe(ctx, audio, voice.FormatM4A)

    // 3. Orchestrator 呼出（通常のテキストと同じ）
    req := orchestrator.ProcessMessageRequest{
        SessionID:   sessionID,
        Channel:     "line",
        ChatID:      event.Source.UserID,
        UserMessage: text,
    }
    resp, _ := h.orchestrator.ProcessMessage(ctx, req)

    // 4. TTS（オプション: 音声モード有効時）
    audioResp, _ := h.synthesizer.Synthesize(ctx, resp.Response, opts)

    // 5. 音声メッセージとして返信
    h.sender.SendAudioReply(ctx, event.ReplyToken, audioResp)
}
```

## 7. main.go 配線パターン

### 7.1 現行の LINE 配線

```go
// cmd/picoclaw/main.go（現行、簡略化）
lineHandler := line.NewHandler(orch, cfg.LineChannelSecret, cfg.LineChannelToken)
mux.Handle("/webhook", lineHandler)
```

### 7.2 マルチチャネル配線（将来）

```go
func cmdRun() {
    // ... 既存の Orchestrator 構築 ...

    // チャネルアダプター起動
    var adapters []adapter.ChannelAdapter

    // LINE（HTTP webhook — 既存）
    if cfg.Channels.Line.Enabled {
        lineHandler := line.NewHandler(orch, cfg.Channels.Line.Secret, cfg.Channels.Line.Token)
        mux.Handle("/webhook", lineHandler)
        // LINE は HTTP サーバーに組み込みなので ChannelAdapter として Start しない
    }

    // Discord（WebSocket — 新規）
    if cfg.Channels.Discord.Enabled {
        discordHandler, err := discord.NewHandler(orch, cfg.Channels.Discord.BotToken)
        if err != nil {
            log.Fatalf("Discord init failed: %v", err)
        }
        adapters = append(adapters, discordHandler)
    }

    // Slack（Socket Mode — 新規）
    if cfg.Channels.Slack.Enabled {
        slackHandler, err := slack.NewHandler(orch, cfg.Channels.Slack.BotToken, cfg.Channels.Slack.AppToken)
        if err != nil {
            log.Fatalf("Slack init failed: %v", err)
        }
        adapters = append(adapters, slackHandler)
    }

    // 各アダプターをバックグラウンド起動
    for _, a := range adapters {
        a := a
        go func() {
            if err := a.Start(ctx); err != nil {
                log.Printf("[%s] adapter stopped: %v", a.Name(), err)
            }
        }()
    }

    // HTTP サーバー起動（LINE + Health）
    srv := &http.Server{Addr: addr, Handler: mux}
    // ...

    // Graceful shutdown
    <-sigCh
    for _, a := range adapters {
        a.Stop()
    }
    srv.Shutdown(ctx)
}
```

## 9. セッション ID 規約

セッション ID の統一フォーマット:

```
{YYYYMMDD}-{channel}-{scope}-{identifier}
```

| チャネル | パターン | 例 |
|---------|---------|-----|
| LINE | `YYYYMMDD-line-{userID}` | `20260306-line-U1234abcd` |
| Discord DM | `YYYYMMDD-discord-dm-{userID}` | `20260306-discord-dm-123456789` |
| Discord Server | `YYYYMMDD-discord-{guildID}-{channelID}` | `20260306-discord-111-222` |
| Slack DM | `YYYYMMDD-slack-dm-{userID}` | `20260306-slack-dm-U0ABC` |
| Slack Channel | `YYYYMMDD-slack-{teamID}-{channelID}` | `20260306-slack-T01-C01` |
| Slack Thread | `YYYYMMDD-slack-{teamID}-{channelID}-{threadTS}` | `20260306-slack-T01-C01-1234.5678` |
| Voice Local | `YYYYMMDD-voice-local-{userID}` | `20260306-voice-local-default` |

日次カットオーバー（`YYYYMMDD` プレフィックス）により、セッションは自然にリセットされる（既存仕様と同一）。

## 10. テスト戦略

### 10.1 テストレベル

| レベル | 対象 | 方式 | ビルドタグ |
|--------|------|------|----------|

| Unit | メンション除去 | テーブル駆動 | なし |
| Integration | Handler → 実 Orchestrator (mock LLM) | httptest / mock WebSocket | なし |
| E2E | 実 Discord/Slack Bot 接続 | 実 API | `e2e` |
